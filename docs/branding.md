# Branding Guidelines – CyberAi Terminal

## Brand Identity

### Official Name
**CyberAi Terminal** (formerly GitHub Mobile Terminal)

### Tagline
"Control your repository from anywhere"

### Alternate Taglines
- "Terminal-style automation for GitHub"
- "Operator-grade repository management"
- "Command your code from the cloud"

## Brand Architecture

### Organizational Hierarchy
- **Organization**: CyberAi – Enterprise automation platform
- **AI Engine**: SmartBrain – Intelligent multi-agent orchestration
- **Product**: Terminal – Command-line automation interface
- **Platform**: GitHub – Integration platform

## Visual Identity

### Logo Usage
- Primary logo: "CyberAi Terminal" wordmark
- Icon: Terminal prompt symbol (`>_`)
- Color scheme: See color palette below

### Color Palette

**Primary Colors**
- Terminal Green: `#00FF00` - Success states, primary actions
- Dark Background: `#0D1117` - Primary background (GitHub dark theme)
- Light Text: `#C9D1D9` - Primary text

**Secondary Colors**
- Command Blue: `#58A6FF` - Links, interactive elements
- Warning Yellow: `#F0E68C` - Warnings, attention needed
- Error Red: `#F85149` - Errors, critical states
- Success Green: `#3FB950` - Success confirmations

**Accent Colors**
- CyberAi Purple: `#8B5CF6` - Brand accent
- SmartBrain Blue: `#3B82F6` - AI/ML features

### Typography

**Primary Font**: Monospace (for terminal aesthetic)
- Recommended: JetBrains Mono, Fira Code, SF Mono
- Fallback: Consolas, Monaco, monospace

**Secondary Font**: Sans-serif (for documentation)
- Recommended: Inter, San Francisco, Segoe UI
- Fallback: system-ui, -apple-system, sans-serif

## Brand Voice

### Tone
- **Professional** but approachable
- **Technical** but clear
- **Confident** but not arrogant
- **Helpful** and supportive

### Writing Style
- Use active voice
- Be concise and direct
- Use terminal/command-line metaphors
- Emphasize automation and efficiency

### Example Phrases
✅ DO:
- "Execute commands directly in PRs"
- "Automate your workflow"
- "Terminal brings operator-grade tools"
- "Command your repository with confidence"

❌ DON'T:
- "Maybe try using our tool"
- "We think this might help"
- "Possibly automate"
- "Terminal is kind of useful"

## Product Positioning

### Primary Value Proposition
Terminal is the command-line interface for GitHub automation, enabling developers to execute powerful repository operations directly from pull request comments.

### Target Audience
- **Primary**: DevOps engineers, site reliability engineers, platform teams
- **Secondary**: Full-stack developers, engineering managers
- **Tertiary**: Open source maintainers, code reviewers

### Key Differentiators
1. **Command-line interface in PRs** - Familiar terminal experience
2. **No context switching** - Work directly in GitHub
3. **Operator-grade automation** - Production-ready workflows
4. **CyberAi ecosystem integration** - Part of larger automation platform

## Usage Guidelines

### Naming Conventions

**Correct Usage:**
- CyberAi Terminal (full name)
- Terminal (when context is clear)
- terminal (in code/commands)
- `/terminal` (command prefix)

**Incorrect Usage:**
- ~~Terminal.ai~~
- ~~GitHub Terminal~~
- ~~CyberAI Terminal~~ (wrong capitalization)
- ~~Cyberai Terminal~~ (missing capital)

### Command Formatting

Always format commands in code blocks:
```
/terminal help
/terminal status
/terminal scan
```
